## **Plugins**

You need to have a fair grasp of plugins to maximally enjoy BTCPay Server as a user. 

This page is dedicated to make you extend the functionalities of your Bitcoin server, by using the most relevant plugins for you to build and scale. 

## **What is a plugin?**

A plugin is a simple tool that extends the capacity of an application. Note that the application can exist and function very well without any plugin.

Here is an example to get what a plugin means:

Imagine you order fried rice and chicken at a restaurant. Then you thought of hitting your fiber target for the day, and requested coleslaw. 

In the instant case, fried rice is the main course. Coleslaw, on the other hand, is a *plugin* to make your fried rice more delicious and also have more fiber. 

### **Plugins on BTCPay Server**

Now that you get the concept, let’s bring it up to the context of BTCPay Server. 

BTCPay is a secure infrastructure for you to accept Bitcoin payment; this is the core offering. Depending on what you want, you can enable plugins to add more functionalities. 

Plugins are more like “extensions” that make you do more things with the BTCPay. Examples of current plugins include the BTCPay Shopify you can connect to Shopify stores to transact in Bitcoin. 

No need to write a single line of code as a user. Click, activate, and use the plugin. 


## **Plugin Availability **

There are currently so many plugins on BTCPay Server. But you might wonder, “if there are so many plugins, how come I’m seeing 3 or 4?”

That’s a valid observation. 

The answer to that is this: you will only see as many plugins as your server admins enable. 

It is only the enabled plugins that can be seen and used. 

Has there been any plugin you’ve wanted to use but are not seeing? You should reach out to your server admin for the same. 


## **Discover Plugins **

You don’t have to do guesswork to know all the plugins in the ecosystem at the moment.  

We have a public [list you can check up here](https://plugin-builder.btcpayserver.org/public/plugins/). 

Whether you’re an e-commerce founder or blogger, there are extensions to facilitate your experience on BTCPay. 

For example, bloggers that use Ghost can simply integrate the [Ghost plugin](https://plugin-builder.btcpayserver.org/public/plugins/ghostplugin). Similarly, you can enable customers to pay with Lightning by integrating the [Boltz plugin.](https://plugin-builder.btcpayserver.org/public/plugins/boltz) 

At the moment, there are 4 filters on the plugins page:

* The Most Rated: To check out plugins based on ecosystem recommendations
* Relevance: A filter that prioritizes plugins that are more peculiar to your objectives
* Most Recent: Use this to get a hold of new plugins
* Alphabetica: See all the plugins listed from A to Z

## **Third-party Risks** 
If you check through the list of all the plugins, you will realize there are different entities behind them. That is to say, BTCPay did not create any plugin. Instead, third-party developers did of their own accord.

On this note, you should bear in mind that not everyone builds open-source plugins for safer Bitcoin payment experiences.

Meaning, plugins carry inherent security risk unless you deobfuscate and assess the code behind it. Thus, you should be very careful and only use these plugins at your risk.


## **How to Rate Plugins **

The ecosystem will discover and use plugins based on recommendations. This is why it is important you drop reviews and ratings for any plugin you might have used with BTCPay Server.

These are some pointers on what to base your review on:

* How easy was the plugin to implement its objectives?
* Did you have a good experience using it?
* What’s the major blocker that made you disable it?
* What do you think can be improved?
* On a scale of 1-5, how will you rate it?
